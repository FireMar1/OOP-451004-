# Library Manager (BSON) — Labs 4–6

This project is a Java (Swing) application that manages a hierarchy of `LibraryItem` objects and saves/loads them using a custom BSON format.

## Lab 4 — Plugins (Hierarchy)

### What is pluggable
- **New item types** can be added via JAR plugins implementing `com.library.plugin.PluginDescriptor`.
- Each item plugin provides:
  - a BSON deserializer (for loading)
  - a dynamic Add/Edit form registration (for UI)

### How plugins are loaded
- Automatically at startup from the **classpath**
- And dynamically from the **`plugins/` folder** located next to the running JAR (or `./plugins` when running from IDE)
- Also via UI: **Plugins → Load plugin JAR…** (copies JAR into `plugins/` and reloads)

### Demo plugin JARs (generated by this project)
Running `mvn -DskipTests clean verify` also creates external plugin JARs in:
- `library-bson/plugins-dist/`

Files:
- `library-bson-plugin-videolecture.jar` (new item type)
- `library-bson-plugin-processors.jar` (processing plugins for Settings)
- `library-bson-plugin-legacy.jar` (legacy plugin for Adapter demo)

## Lab 5 — Plugins (Functional processing)

Functional plugins implement `com.library.plugin.ProcessingPlugin` and can transform the loaded/saved structures:
- **BEFORE_SAVE**: executed right before writing to file
- **AFTER_LOAD**: executed right after reading from file

UI:
- **Settings → Before save / After load** shows checkboxes depending on loaded plugins.

## Lab 6 — Patterns

### Adapter pattern
Legacy/foreign plugins can implement `com.library.legacy.LegacyProcessor` (a different interface).
The app automatically **adapts** them to `ProcessingPlugin` inside `com.library.plugin.PluginManager`.

### Strategy pattern
Each `ProcessingPlugin` is a **strategy** for transforming the list of items.
The app composes a pipeline based on enabled checkboxes in the Settings menu.

### Facade (bonus)
`PluginManager` provides a single entry point to plugin loading and registration.

## Build & Run

From `library-bson/`:

```bash
mvn -DskipTests clean verify
java -jar target/library-bson.jar
```

Then place external plugin JARs from `plugins-dist/` into `plugins/` (next to the JAR) and use **Plugins → Reload**.
